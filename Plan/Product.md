# PRODUCT.md — CATalyst

---

# 🧠 1. Product Overview

CATalyst is a focused CAT preparation web application designed to help aspirants improve performance through structured practice and mistake correction.

It is not a coaching platform.  
It is not a content-heavy test series.

It is a personal performance improvement system.

---

# 🎯 2. Problem Statement

CAT aspirants face a consistent gap between effort and improvement.

### Observed Behavior:
- Students solve questions regularly
- They attempt mocks and practice sets
- They receive scores and analytics

---

### Core Problems:

#### ❌ 1. Repetition of Mistakes
Students repeatedly make the same errors across topics without realizing patterns.

---

#### ❌ 2. Lack of Structured Error Tracking
Although “error logs” are recommended:
- They are manual
- Time-consuming
- Inconsistent

As a result, most students do not maintain them.

---

#### ❌ 3. No Actionable Feedback
Existing platforms provide:
- Accuracy
- Scores
- Percentiles

But do not answer:
> “What exactly should I fix next?”

---

# 💡 3. Solution — CATalyst

CATalyst introduces a structured system:

> Practice → Capture Error → Identify Pattern → Improve

---

## Core Value:

- Reduce repeated mistakes
- Make error tracking effortless
- Provide clarity on weak areas
- Turn practice into improvement

---

# ⚙️ 4. How CATalyst Works

---

## 🔹 Step 1: Practice

User:
- selects topic / mode
- solves questions

System:
- tracks every attempt

---

## 🔹 Step 2: Attempt Tracking

For every question:
- selected answer stored
- correctness recorded
- time tracked

---

## 🔹 Step 3: Error Logging (Core Feature)

If answer is incorrect:

User selects:
- conceptual mistake  
- silly mistake  
- guess  
- time pressure  

Optional:
- short note

👉 Stored automatically

---

## 🔹 Step 4: Data Collection

System builds:

- accuracy by topic  
- error type distribution  
- performance trends  

---

## 🔹 Step 5: Feedback to User

User sees:

- weak topics  
- accuracy  
- mistake patterns  

---

# 🚀 7. Product Philosophy

---

### 1. Simplicity First
Avoid complex features that reduce usage.

---

### 2. Actionable Insight > Raw Data
User should know what to fix, not just see numbers.

---

### 3. Behavior Over Features
If users don’t use a feature, it doesn’t exist.

---

### 4. Build → Test → Improve
Product evolves based on real user feedback.
