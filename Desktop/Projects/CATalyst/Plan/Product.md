1. What this product is

CATalyst is not a practice app.
It is a mistake-fixing system for CAT aspirants.

Most apps help users solve more questions.
CATalyst helps users stop repeating mistakes.

⸻

2. Core Problem

Users:

* practice a lot
* but repeat the same mistakes
* don’t know why they are wrong
* don’t fix patterns

👉 Result: score stays stuck

⸻

3. Core Solution

CATalyst creates a loop:

Solve → Realize → Diagnose → Fix → Reinforce → Return

Instead of:

Solve → Next → Forget

⸻

4. Core Concept (IMPORTANT)

This product runs on:

👉 “Mistake Pressure System”

* Makes mistakes visible
* Shows cost (marks lost)
* Forces acknowledgment
* Pushes user to fix immediately

⸻

5. User Types

1. New User

* No data
* Goal: discover weak areas

2. Returning User

* Has mistakes logged
* Goal: fix pending mistakes

⸻

6. Key Features

1. Practice Mode

* Normal question solving
* Tracks mistakes
* Adds behavioral hint:
    * “Focus: notice where you go wrong”

⸻

2. Wrong Answer System (CRITICAL)

After every wrong answer:

User must:

* feel the mistake
* tag the reason:
    * Concept
    * Calculation
    * Misread
    * Guess

👉 No passive learning

⸻

3. Session Diagnosis

At end:

* “You made X mistakes”
* Breakdown
* Highlight:
    * “This is why your score is stuck”

👉 Push to Fix Mode

⸻

4. Fix Mode (Core Engine)

Phase 1:

* Solve past mistakes again
* Label:
    * “You got this wrong before”

Phase 2:

* Strengthening weak topics
* New questions

⸻

5. Error Log (Pain Board)

Not a table.

Shows:

* biggest issue
* cost in marks
* list of mistakes

CTA:
👉 “Fix this now”

⸻

6. Loop System

User never finishes clean.

Always:

* “X mistakes still remain”

👉 creates return behavior

⸻

7. UX Philosophy

Rules:

1. Pain > Action > Everything else
2. One CTA per screen
3. Red = mistake only
4. Blue = action only
5. No “nice dashboard symmetry”

⸻

8. Emotional States by Screen

Screen	Emotion
Login	Curiosity
Dashboard	Pressure
Practice	Focus
Wrong Answer	Sting
Session Complete	Realization
Fix Mode	Control
Error Log	Confrontation

⸻

9. What this product is NOT

* Not a question bank
* Not a mock test platform
* Not a dashboard tool

⸻

10. Success Metric

User behavior:

* comes back to fix mistakes
* reduces repeat errors
* improves accuracy

⸻

11. Current Version

* V1 = basic working app
* V2 = psychological + system-driven product

⸻

12. Build Philosophy

* fast iteration
* AI-assisted development (Claude Code)
* focus on behavior, not features
